# 低活跃书籍唤醒清单 — 需求说明文档

## 1. 需求背景

用户在使用读书笔记应用的过程中，往往会积累大量"半死不活"的书籍：
- 开了头但没坚持读完的书
- 加入想读却一直没开始的书
- 读完了却没做总结、知识没闭环的书

为了帮助用户"唤醒"这些沉睡的书籍，补齐知识闭环，特开发 **低活跃书籍唤醒清单（待回访书单）** 功能。

---

## 2. 功能目标

- 自动识别三类低活跃书籍，形成待回访列表
- 提醒用户重新拾起停滞的阅读、启动尘封的想读书、补写已读书籍的总结
- 促进阅读习惯养成，提升知识留存率

---

## 3. 三类低活跃书籍定义

### 3.1 在读停滞（长期在读却无新增笔记）

- **状态条件**：书籍阅读状态为 `READING`（在读）
- **时间条件**：超过阈值天数（默认 30 天）没有新增笔记
- **典型场景**：用户开了一本书，读了几章，记了几条笔记，然后就搁下了
- **唤醒价值**：鼓励用户捡起来继续读，别让之前的努力白费

### 3.2 想读尘封（长期想读未启动）

- **状态条件**：书籍阅读状态为 `WANT_TO_READ`（想读）
- **时间条件**：加入想读清单的时间超过阈值天数（默认 30 天）
- **进度条件**：阅读进度为 0 或空（表示从未开始）
- **典型场景**：用户看到推荐随手加入想读，然后就忘了
- **唤醒价值**：推动用户做决定 —— 要么开始读，要么移出想读清单

### 3.3 已读无总结（已读但无总结）

- **状态条件**：书籍阅读状态为 `READ`（已读）
- **总结条件**：没有任何被判定为"总结"的笔记
- **典型场景**：书读完了，也记了不少摘抄笔记，但没有整本书的总结感悟
- **唤醒价值**：引导用户输出总结，完成「输入 → 加工 → 输出」的知识闭环

---

## 4. "总结笔记"判定规则

"总结笔记"采用 **标题关键词 + 标签** 的双重识别策略，满足任一条件即视为总结笔记：

### 4.1 标题关键词匹配

笔记标题包含以下关键词之一（不区分大小写）：
- 总结
- 读后感
- 书评
- 感悟
- 读书心得

### 4.2 标签匹配

笔记被打上了名称为 **"总结"** 的标签（不区分大小写）

### 设计说明

- 两种识别方式互为补充，用户无论习惯用哪种方式都能被正确识别
- 标题关键词识别是"开箱即用"的，不需要用户额外操作
- 标签识别更精准，适合有整理习惯的用户
- 规则可扩展，未来可根据实际使用情况调整关键词或添加其他识别维度

## 5. 功能设计

### 5.1 后端 API

#### 接口地址

```
GET /api/activity/wake-up-list
```

#### 请求参数

| 参数名 | 类型 | 必填 | 默认值 | 说明 |
|--------|------|------|--------|------|
| daysThreshold | int | 否 | 30 | 低活跃判定阈值（天） |

#### 响应结构

```json
{
  "stagnantReading": [
    {
      "book": { "id": 1, "title": "书名", "author": "作者", "status": "READING", "progress": 35, "noteCount": 5, ... },
      "daysInactive": 45,
      "reason": "已连续 45 天无新增笔记",
      "category": "stagnantReading"
    }
  ],
  "neglectedWantToRead": [...],
  "readWithoutSummary": [...],
  "totalCount": 10
}
```

#### 响应字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| stagnantReading | Array | 在读停滞书籍列表 |
| neglectedWantToRead | Array | 想读尘封书籍列表 |
| readWithoutSummary | Array | 已读无总结书籍列表 |
| totalCount | int | 待回访书籍总数 |
| book | Object | 书籍基本信息（同书籍详情接口） |
| daysInactive | long | 无活跃天数 |
| reason | String | 唤醒原因文案（人性化描述） |
| category | String | 分类标识 |

---

### 5.2 前端页面

#### 页面路径

```
/wake-up
```

#### 页面入口

顶部导航栏新增 **"待回访"** 菜单项，位于"书籍库"之后。

#### 页面结构

1. **页面头部**
   - 标题：待回访书单
   - 副标题：识别长期低活跃的书籍，提醒你补齐知识闭环
   - 阈值切换：7天 / 15天 / 30天 / 60天 / 90天

2. **统计卡片**（三张，点击可快速切换到对应分类）
   - 在读停滞数量
   - 想读尘封数量
   - 已读无总结数量

3. **分类标签页**
   - Tab 切换三类列表
   - 每个 Tab 显示分类说明和书籍卡片
   - 卡片包含：封面、书名、作者、状态、进度/笔记数、无活跃天数、唤醒原因、操作按钮

4. **空状态**
   - 全部活跃时显示祝贺文案

---

## 6. 交互设计

### 6.1 阈值切换

- 提供 7/15/30/60/90 天五档阈值
- 默认选中 30 天
- 切换阈值后自动刷新列表

### 6.2 列表排序

- 按 **无活跃天数倒序** 排列
- 最需要关注的书籍排在最前面

### 6.3 操作按钮

| 分类 | 按钮文案 | 跳转目标 |
|------|---------|---------|
| 在读停滞 | 继续阅读 | 书籍详情页 |
| 想读尘封 | 开始阅读 | 书籍详情页 |
| 已读无总结 | 写总结 | 新建笔记页（预设标题可引导为"XX 读后感"） |

## 8. 后续可扩展方向

- **推送提醒**：结合定时任务，定期通过邮件/站内信推送待回访清单
- **智能推荐**：根据用户阅读习惯，优先推荐最可能捡起来的书
- **成就系统**：连续回访、读完"在读停滞"的书可获得徽章
- **总结模板**：为"写总结"提供模板引导（如三段式总结法）
- **批量操作**：支持批量将想读书籍标记为"放弃"
